🤔 What Are We Solving?

The Big Question:

How likely is a fire to spread completely across a forest?

  • 🌲 Forest = Grid of squares (each might have a tree)
  • 🔥 Fire starts on the left side
  • ⬆️⬇️⬅️➡️ Fire spreads to neighboring trees (up, down, left, right)
  • 🎯 Goal: Does fire reach the right side?
  • 🔢 We test this thousands of times with different tree densities

⚡ Two Ways to Run the Simulation

🐌 Sequential (Slow)

  • Uses 1 computer thread
  • Runs one trial at a time
  • Simple but slow

🚀 Parallel (Fast)

  • Uses 8 computer threads
  • Runs multiple trials at once
  • Much faster!

Key Idea: Since each trial is independent, we can run many at the same time!

5 / 10

✨ The Magic Number: 55%

🔥 The "Percolation Threshold"

Around 55% tree density, something amazing happens!

  • 📉 Below 55%: Fire almost never spreads across (0-0.1%)
  • 📈 Above 55%: Fire spreads much more often (6% → 86% → 100%)
  • 🧩 Why? Trees start forming connected paths across the forest
  • 🌉 Think of it like: Building bridges - you need enough pieces to connect both sides
This sudden jump is a well-known phenomenon in science called "percolation"!
8 / 10

🚀 How Much Faster Is Parallel?

Speed-up Results:

3.2× to 6.6× faster!
  • 🎯 Best Case: 6.6× faster (density 45% and 60%)
  • 📊 Average: About 5× faster
  • 💪 Efficiency: 70% of perfect speed-up (8 threads = 8× would be perfect)
  • 🔥 CPU Usage: Sequential uses 5%, Parallel uses 30%

Bottom Line: Parallel programming made our simulation 5 times faster on average!

9 / 10

🎓 What We Learned

  • 🔥 Fire behavior: There's a magic threshold around 55% tree density
  • ⚡ Parallel power: Using multiple threads can make simulations 5× faster
  • ✅ Accuracy: Parallel and sequential versions give the same results
  • 🎲 Monte Carlo: Perfect for parallelization because each trial is independent
  • ☕ Java tools: ExecutorService and LongAdder make parallel programming easier

Real-world impact: This approach can be used for weather prediction, disease spread, financial modeling, and more!
